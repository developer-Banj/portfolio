About Me

API Security is not just what I do —
it's how I think.

I'm David Banjo, a cybersecurity practitioner with a foundation in enterprise networking who has narrowed my focus to what I believe is the most critical and under-tested attack surface in modern software: APIs.

Coming from hands-on network engineering (Cisco ISE, 802.1X/MAB, Catalyst 9K, ASR 9000 on live MTN Nigeria infrastructure), I understand both the infrastructure layer and the application layer — which gives me a unique vantage point when assessing API security.

My current focus is drilling deep into the OWASP API Security Top 10, JWT attack chains, broken object-level authorization, and business logic flaws — and I document everything through write-ups and technical blog posts.

Longer term, I'm building a pathway to help African talent break into cybersecurity careers.

API Testing Methodology

A structured, repeatable approach to API security testing — click any stage to explore what it involves.

01

Reconnaissance

Passive and active discovery of API endpoints, documentation, and attack surface. Includes Google dorking, Shodan queries, GitHub secret leaks, and JS file analysis.

ShodanGoogle DorksFFuFtruffleHog
02

Documentation Review

Analysis of Swagger/OpenAPI specs, Postman collections, and API documentation to identify exposed endpoints, insecure parameters, and design-level vulnerabilities.

Swagger UIPostmanOpenAPI
03

Authentication Testing

JWT attacks (alg:none, key confusion, weak secrets), OAuth flow abuse, API key enumeration, and credential stuffing against authentication endpoints.

jwt_toolBurp SuiteHashcat
04

Authorization Testing

BOLA and BFLA testing across all object types, horizontal and vertical privilege escalation, and tenant isolation bypass in multi-tenant APIs.

Burp SuitePostmanAuthMatrix
05

Input Validation

SQL injection, NoSQL injection, XXE, SSRF, and command injection via API parameters. Mass assignment and parameter pollution testing.

SQLMapBurp SuiteOWASP ZAP
06

Business Logic Testing

Identifying logic flaws unique to the application — price manipulation, coupon abuse, workflow bypass, and race conditions that automated tools cannot detect.

Rate Limiting & DoS
